# API Module

Versioned public REST API exposing HackGreenville events and organizations as JSON.

## Routes
- `GET /api/v0/events`, `GET /api/v0/orgs` — legacy flat format (Drupal-era compatibility)
- `GET /api/v1/events`, `GET /api/v1/organizations` — modern paginated format
- Docs: `/docs/api` (auto-generated by Scribe)

## Testing
- Run module tests: `php artisan test app-modules/api/tests/Feature`
- v0 tests use exact JSON assertions (backward compatibility matters)
- v1 tests use structure/path assertions

## v0 vs v1 Differences
- **v0**: No pagination, flat responses, Drupal-era field names (`field_*`), only returns first tag ID. Exists purely for backward compatibility — do not add features to v0.
- **v1**: Full pagination (`data`/`links`/`meta` envelope), nested resources, rich filtering (date, name, venue, RSVP, is_paid, service, tags), sorting by multiple fields, tags as array of objects.

## Gotchas
- **Scribe mock detection**: `ApiResource` base class detects `scribe:generate` and returns fixed mock values. If you add a new resource, extend `ApiResource` to keep docs stable.
- **`is_paid=null` filter**: v1 accepts string `"null"` as query param and converts it to actual null for filtering events with unknown paid status.
- **Soft-deleted orgs**: Both versions explicitly filter `whereNull('deleted_at')` on the organization relation — events from deleted orgs never appear.
- **No authentication**: All endpoints are public. Rate limiting comes from the `api` middleware group only.
- **Route auto-discovery**: Routes loaded automatically by the Modular library from `routes/api-routes.php` — no manual registration needed.

## Regenerating API Docs
```bash
php artisan scribe:generate
```
Config in `config/scribe.php`. Use `@apiResource` docblock annotations in controllers.

## Config
- `config/events-api.php` — `EVENTS_API_DEFAULT_DAYS` (default: 30) controls how far back events are returned when no date range is specified.
